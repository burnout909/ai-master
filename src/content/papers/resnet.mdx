<StageSection id="intuition" title="① Intuition">

Before ResNet, the community assumed that adding more layers to a neural network always helps — more parameters means more capacity. But in 2015 He et al. showed something disturbing: when you stack plain layers past ~20 deep on CIFAR-10 or ImageNet, **training accuracy gets worse**, not just test accuracy. This is not overfitting — the training error itself rises. The network simply refuses to learn the identity mapping through many layers.

The fix is surprisingly simple: give the network a **shortcut**. Instead of asking each block to directly learn the desired mapping H(x), let it learn the residual F(x) = H(x) − x, then add back the input: H(x) = F(x) + x. If the optimal mapping is close to identity, the residual is near zero — much easier to push towards zero than to fit an identity through stacked weight layers.

<StageSection id="math" title="② Math">

A residual block with input **x** and learned residual function **F**:

<Math display expr="y = F(x,\{W_i\}) + x" />

When the input and output dimensions differ (e.g., when doubling channels or halving spatial size), a linear projection is applied to the shortcut:

<Math display expr="y = F(x,\{W_i\}) + W_s\, x" />

W_s is a 1×1 convolution that matches dimensions. F typically contains two or three conv-BN-ReLU sub-layers; the final ReLU is applied **after** the addition.

The gradient through a residual block is:

<Math display expr="\frac{\partial \mathcal{L}}{\partial x} = \frac{\partial \mathcal{L}}{\partial y}\left(1 + \frac{\partial F}{\partial x}\right)" />

The "+1" term from the skip connection means the gradient can never vanish completely — even if ∂F/∂x ≈ 0, the loss gradient flows unchanged back to x.

<StageSection id="code" title="④ Code (Python)">

Implement a residual block forward pass in NumPy. Use simple matrix multiplication (no actual convolution) to keep it clear — treat each "conv" as a linear layer applied to a flattened input.

<HintFade client:load
  hints={[
    "Save the input before any transformation: identity = x.copy(). Then apply two linear-ReLU layers to get F(x).",
    "The second linear layer should NOT have a ReLU applied to it inside F — ReLU only comes after the addition.",
    "After computing out = W2 @ relu(W1 @ x), add the saved identity: out = out + identity, then apply relu.",
  ]}
  solutionCode={`import numpy as np

def relu(x):
    return np.maximum(0, x)

def residual_block(x, W1, W2):
    identity = x.copy()
    out = relu(W1 @ x)      # conv1 + ReLU
    out = W2 @ out           # conv2 (no ReLU yet)
    out = out + identity     # add shortcut
    return relu(out)         # final ReLU`}
/>
