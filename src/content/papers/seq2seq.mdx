<StageSection id="intuition" title="① Intuition">

Before seq2seq, neural machine translation was largely handled by phrase-based statistical models — pipelines of hand-tuned components that decomposed sentences into fragments, translated each fragment, and stitched them back together. These systems were complex, brittle, and required extensive feature engineering for each language pair.

**Sutskever, Vinyals, and Le's key insight:** a single end-to-end trained LSTM can map a variable-length source sequence to a variable-length target sequence by first **encoding** the entire source into one fixed-size vector **v** (the final hidden state), then **decoding** from that vector token by token. No phrase tables, no alignment models — just two RNNs and a training objective. The paper also uncovered a surprising empirical trick: **reversing the source sequence** boosted BLEU by ~5 points. When the source is reversed, the first source words (which must align to the first target words) end up at the end of the encoding trajectory — just a few LSTM steps from where decoding begins. Shorter "temporal distance" makes the gradient path shorter and alignment easier.

<StageSection id="math" title="② Math">

The seq2seq model defines a conditional probability over the target sequence given the source. The encoder reads source tokens <Math expr="x_1, \ldots, x_{T_{\text{src}}}" /> and produces a single summary vector:

<Math display expr="v = h_{T_{\text{src}}}^{\text{enc}}" />

The decoder then factorises the joint target probability using the chain rule:

<Math display expr="p(y_1, \ldots, y_{T_{\text{tgt}}} \mid x_1, \ldots, x_{T_{\text{src}}}) = \prod_{t=1}^{T_{\text{tgt}}} p\!\left(y_t \mid v,\, y_1, \ldots, y_{t-1}\right)" />

Each factor <Math expr="p(y_t \mid v, y_{<t})" /> is computed by a softmax over the vocabulary applied to the decoder hidden state <Math expr="h_t^{\text{dec}}" />, which evolves as:

<Math display expr="h_t^{\text{dec}} = \text{LSTM}\!\left(h_{t-1}^{\text{dec}},\, [y_{t-1};\, v]\right)" />

Training maximises the log-likelihood over all sentence pairs <Math expr="(X, Y)" /> in the corpus:

<Math display expr="\mathcal{L} = \sum_{(X,Y)} \sum_{t=1}^{T_{\text{tgt}}} \log p\!\left(y_t \mid v,\, y_{<t}\right)" />

At inference time, **beam search** maintains the top-B partial hypotheses at each step, expanding each by all vocabulary tokens and retaining only the B highest-probability continuations.

def greedy_decode(v, W_h, U_e, W_out, embeddings, n_steps):
    """
    v:          (d_h,)  encoder context vector
    W_h:        (d_h, d_h)
    U_e:        (d_h, d_emb)
    W_out:      (vocab, d_h)
    embeddings: (vocab, d_emb)
    n_steps:    int
    """
    h = v.copy()
    y = 0  # BOS index
    outputs = []
    for _ in range(n_steps):
        e = embeddings[y]            # (d_emb,)
        h = np.tanh(W_h @ h + U_e @ e)  # (d_h,)
        logits = W_out @ h           # (vocab,)
        y = int(np.argmax(logits))
        outputs.append(y)
    return outputs
`}
