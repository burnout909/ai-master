<StageSection id="intuition" title="① Intuition">

Before CLIP, the standard recipe for visual recognition was: collect a labelled dataset (ImageNet: 1.28M images, 1000 classes), train a supervised model, ship. This approach is powerful but brittle — if you want to classify a category not in those 1000 labels, you must re-label data and retrain. The label set is a hard ceiling.

CLIP (Contrastive Language-Image Pre-training) breaks that ceiling by learning from **natural language supervision**. Instead of fixed integer labels, each image is paired with a free-form caption scraped from the web. OpenAI assembled 400 million such (image, text) pairs. An image encoder and a text encoder are jointly trained so that matching pairs have high cosine similarity and mismatched pairs have low similarity. At inference time, zero-shot classification requires no additional training: you write text prompts ("a photo of a cat", "a photo of a dog", ...) for any label set, encode them, and pick the label whose embedding is closest to the image embedding.

<PredictThenReveal client:load predictPrompt="Before playing with the widget: if you lower the temperature in a softmax over similarity logits, what happens to the probability distribution? Does the diagonal get sharper or flatter?">
  Lowering temperature divides the logits by a smaller number, making the differences between them larger before the softmax. A larger spread of inputs to softmax produces a sharper (more peaked) distribution. So at low temperature the diagonal entries — the correct image-text matches — dominate strongly, and off-diagonal entries collapse toward zero.
</PredictThenReveal>

Now raise temperature to 1.5: the matrix grays out and all cells look similar — the model is uncertain. This temperature parameter (tau in the paper) is learned during training, not hand-tuned.

<StageSection id="math" title="② Math">

Given a training batch of N (image, text) pairs, CLIP computes two sets of embeddings:

<Math display expr="\mathbf{I} \in \mathbb{R}^{N \times d}, \quad \mathbf{T} \in \mathbb{R}^{N \times d}" />

Both are L2-normalized so that the dot product equals cosine similarity. The logit matrix is:

<Math display expr="\mathbf{L} = \frac{\mathbf{I} \mathbf{T}^\top}{\tau}" />

where tau is a learned (scalar) temperature parameter. The diagonal of L holds the scores for correct pairs; the off-diagonal entries are scores for incorrect pairs. The training loss is symmetric cross-entropy over the rows (image-to-text direction) and columns (text-to-image direction):

<Math display expr="\mathcal{L} = \frac{1}{2}\left[\text{CE}(\mathbf{L},\, \mathbf{y}) + \text{CE}(\mathbf{L}^\top,\, \mathbf{y})\right], \quad \mathbf{y} = [0, 1, \ldots, N-1]" />

where CE(L, y) treats each row i as a softmax classification problem with correct class y_i = i, and CE(L^T, y) does the same column-wise.

<StageSection id="code" title="④ Code (Python)">

Implement CLIP's symmetric contrastive loss in NumPy. Given I (N x d) and T (N x d), both already L2-normalized, compute the scalar loss. Labels are the diagonal indices [0, 1, ..., N-1].

<HintFade client:load
  hints={[
    "Start by computing the logit matrix: logits = I @ T.T / temperature. Then define labels = np.arange(N). You need cross-entropy in two directions: rows (image-to-text) and columns (text-to-image).",
    "For cross-entropy: softmax each row, then -log(prob[i, labels[i]]). Equivalently: log_softmax = logits - log(sum(exp(logits), axis=1, keepdims=True)); ce = -mean(log_softmax[range(N), labels]). Apply this to logits (rows) and logits.T (columns).",
    "Full solution: logits = I @ T.T / temperature; labels = np.arange(len(I)); def ce(l): ls = l - np.log(np.sum(np.exp(l - l.max(1,keepdims=True)), axis=1, keepdims=True)) - l.max(1,keepdims=True); return -np.mean(ls[range(len(l)), labels]); return (ce(logits) + ce(logits.T)) / 2",
  ]}
  solutionCode={`import numpy as np

def clip_loss(I, T, temperature=0.07):
    N = len(I)
    logits = I @ T.T / temperature
    labels = np.arange(N)

    def cross_entropy(l):
        # numerically stable log-softmax
        l_shifted = l - l.max(axis=1, keepdims=True)
        log_sum_exp = np.log(np.sum(np.exp(l_shifted), axis=1, keepdims=True))
        log_probs = l_shifted - log_sum_exp
        return -np.mean(log_probs[np.arange(len(l)), labels])

    return (cross_entropy(logits) + cross_entropy(logits.T)) / 2`}
