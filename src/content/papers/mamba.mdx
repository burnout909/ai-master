<StageSection id="intuition" title="① Intuition">

Transformers dominate sequence modeling, but they carry a fundamental cost: self-attention is O(L²) in sequence length L. Every new token must attend to every previous token. At L = 4096 that is 16 million dot products per layer; at L = 100k it becomes prohibitive. The KV-cache scales linearly with context, but the attention computation itself scales quadratically — making very long-context inference slow and expensive.

State space models (SSMs) offer a linear-time alternative. An SSM maintains a hidden state `h_t` that summarizes all past inputs, updating it with a simple recurrence: `h_t = A h_(t-1) + B x_t`. The output is `y_t = C h_t`. Because each step touches only h_t and x_t, inference is O(L) — no quadratic explosion. The catch: classical SSMs use fixed matrices A, B, C that do not depend on the input. This makes the dynamics input-agnostic — the model integrates every token with the same weights, blurring relevant content into an undifferentiated running average. It cannot selectively remember an important token or forget irrelevant noise.

Mamba's core insight is to make the SSM parameters **input-dependent**. The step size Delta, the input projection B, and the output projection C all become functions of x_t. Delta_t controls how much the current token is "written" into the hidden state: a large Delta means absorb strongly, a small Delta near zero means skip this token. This gives the model a selective memory: it can latch onto salient content and discard noise — behavior that classical SSMs simply cannot produce.

<StageSection id="math" title="② Math">

Mamba starts from the continuous-time state space model:

<Math display expr="h'(t) = A\,h(t) + B\,x(t), \quad y(t) = C\,h(t) + D\,x(t)" />

where h(t) is the hidden state vector, x(t) the scalar input, and A, B, C, D are matrices/vectors. To apply this on discrete token sequences, Mamba discretizes via the **zero-order hold** (ZOH) rule using a learned step size Delta:

<Math display expr="\bar{A} = \exp(\Delta A), \qquad \bar{B} \approx \Delta B" />

This gives the discrete recurrence used at every time step:

<Math display expr="h_t = \bar{A}\,h_{t-1} + \bar{B}\,x_t, \qquad y_t = C\,h_t" />

So far this is a standard discretized SSM. Mamba's key move: make Delta, B, and C **functions of the current input x_t** via learned linear projections:

<Math display expr="B_t = S_B(x_t),\quad C_t = S_C(x_t),\quad \Delta_t = \text{softplus}(S_\Delta(x_t))" />

where S_B, S_C, S_Delta are linear layers. Because Delta enters the ZOH formula, a large Delta_t means a large Ā_t and a large B̄_t — the model strongly absorbs x_t into the state. A small Delta_t near zero means Ā_t approaches the identity (perfect memory of the past) and B̄_t approaches zero (ignore the current input). This is content-aware gating built into the scan kernel itself.

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def selective_scan(x):
    """
    x: (L,) input sequence
    returns: h (L,) hidden states, y (L,) outputs with C=1

    L = len(x)
    hs = []
    h = 0.0

    y = hs  # C = 1
    return hs, y`}
