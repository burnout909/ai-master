<StageSection id="intuition" title="① Intuition">

General-purpose agents struggle in open-ended environments like Minecraft because they face two compounding problems. First, tasks have long horizons: surviving and thriving requires hundreds of interdependent steps — chop wood, craft tools, mine stone, smelt iron — none of which can be solved with a single prompt. Second, every skill is potentially useful again: if an agent learns to mine coal in episode 3, it shouldn't re-derive that procedure from scratch in episode 47. Prior embodied agents either hard-coded a fixed skill set (brittle) or used raw action traces (non-reusable when the world changes). The result: agents that plateau early and never compound their knowledge.

Voyager's insight is to treat a growing **skill library** as the agent's long-term memory and to write skills as executable code rather than recorded action sequences. Code generalizes: `mineWood(count=10)` works regardless of the agent's exact starting position, whereas a trace of button presses replays incorrectly whenever the world layout differs. Voyager builds three mutually reinforcing modules on GPT-4: (1) an **automatic curriculum** that proposes progressively harder tasks, (2) a **skill library** that stores verified JavaScript/Mineflayer code snippets keyed by natural-language descriptions and retrieved by embedding similarity, and (3) an **iterative prompting** loop that writes, executes, receives environment feedback, and self-improves the code until it passes.

<StageSection id="math" title="② Math">

The skill library `M` stores entries of the form `(d_i, c_i, e_i)` where `d_i` is a natural-language description, `c_i` is the executable code, and `e_i = \text{embed}(d_i)` is the embedding vector. Retrieval for a query description `q` selects the top-k entries by cosine similarity:

<Math display expr="\text{top}_k(q,\, M) = \underset{k\text{-largest}}{\operatorname{argmax}}_{i} \;\frac{e_i \cdot \text{embed}(q)}{\|e_i\|\;\|\text{embed}(q)\|}" />

The automatic curriculum generates the next task `t*` by maximizing expected exploration conditioned on full agent state:

<Math display expr="t^* = \text{LLM}_{\text{curriculum}}\!\bigl(\text{state},\; C_{\text{done}},\; C_{\text{fail}}\bigr)" />

where `state` encodes current inventory and nearby blocks, `C_{\text{done}}` is the set of completed tasks, and `C_{\text{fail}}` is the set of recently failed tasks. This is not an explicit optimization — the LLM acts as a learned heuristic that has absorbed exploration principles during pretraining.

Given a list of skill dicts (each with `"description"` and `"embedding"` as a `np.ndarray`), a `query_embedding` (`np.ndarray`), and an integer `k`, return the top-k skills sorted by descending cosine similarity to the query. <HintFade client:load
  hints={[
    "Cosine similarity between two vectors a and b is `dot(a, b) / (norm(a) * norm(b))`. Compute it for each skill's embedding against the query_embedding using a loop or a vectorised stack.",
    "Stack all embeddings into a matrix E of shape (n, d) with `np.stack([s['embedding'] for s in skills])`. Then compute all cosine similarities at once: `sims = E @ query_embedding / (np.linalg.norm(E, axis=1) * np.linalg.norm(query_embedding))`. Use `np.argsort(sims)[::-1]` to get indices in descending order.",
    "Full solution: stack embeddings, compute sims, argsort descending, slice to min(k, n), return `[skills[i] for i in top_indices]`. Edge case: if skills is empty return [].",
  ]}
  solutionCode={`import numpy as np

def retrieve_top_k(skills, query_embedding, k):
    """
    skills: List[Dict] with keys 'description' (str) and 'embedding' (np.ndarray)
    query_embedding: np.ndarray of the same dimension as skill embeddings
    k: int — number of top skills to return
    Returns: List[Dict] of up to k skills sorted by descending cosine similarity
    """
    if not skills:
        return []
    E = np.stack([s["embedding"] for s in skills])          # (n, d)
    q = query_embedding / np.linalg.norm(query_embedding)   # unit query
    norms = np.linalg.norm(E, axis=1, keepdims=True)        # (n, 1)
    E_norm = E / norms                                       # unit skill vecs
    sims = E_norm @ q                                        # (n,)
    top_indices = np.argsort(sims)[::-1][:k]
    return [skills[i] for i in top_indices]`}
