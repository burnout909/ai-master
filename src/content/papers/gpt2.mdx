<StageSection id="intuition" title="① Intuition">

Before GPT-2, the standard playbook for NLP was **pretrain then fine-tune**: train a language model on unlabeled text, then add a task-specific head and supervised training for each downstream task. GPT-2's central claim is more radical: a large enough language model trained on diverse web text can perform many tasks **zero-shot** — with no gradient updates — simply by conditioning the model on a natural-language prompt that describes the task.

The key ingredients are a **decoder-only Transformer** (causal, left-to-right attention), 1.5 billion parameters, and 40 GB of high-quality web text (WebText). At inference time, the task is framed as text: "Translate English to French: cheese =>" or "Q: What is the capital of France? A:". The model completes the prompt without ever seeing labeled examples.

Temperature sampling is central to GPT-2's generation quality. The model outputs a logit vector over the vocabulary; dividing by temperature T before softmax sharpens (low T) or flattens (high T) the resulting distribution.

**Interact with the widget:** set T to 0.3, observe that "the" dominates with very high probability. Then raise T to 1.5 — probability mass spreads across more tokens. This single scalar controls the creativity-coherence tradeoff in every GPT-family model.

<StageSection id="math" title="② Math">

At each generation step, the model produces a logit vector **z** over the vocabulary. Temperature T rescales these logits before the softmax:

<Math display expr="p(x_t \mid x_{<t}) = \text{softmax}\!\left(\frac{\mathbf{z}}{T}\right)_t = \frac{\exp(z_t / T)}{\sum_j \exp(z_j / T)}" />

The **entropy** of the resulting distribution measures its spread:

<Math display expr="H = -\sum_i p_i \log p_i" />

High entropy means the model is uncertain and assigns non-trivial probability to many tokens (creative, diverse). Low entropy means the model is confident and concentrates on a single token (predictable, coherent).

Implement temperature-scaled softmax in NumPy. Given a logit array and a temperature scalar, return a probability distribution. The function should handle T=1 (standard softmax), T close to 0 (near-greedy), and verify that output probabilities sum to 1.

<HintFade client:load
  hints={[
    "Start with numerical stability: subtract the max logit before exponentiating. Then divide all logits by temperature first, then apply the stable softmax.",
    "Structure: scaled = logits / T; then numerically stable softmax: exp(scaled - max(scaled)) / sum(exp(scaled - max(scaled))).",
    "Full implementation: def temperature_softmax(logits, T): scaled = np.array(logits, dtype=float) / T; shifted = scaled - scaled.max(); exps = np.exp(shifted); return exps / exps.sum()",
  ]}
  solutionCode={`import numpy as np

def temperature_softmax(logits, T=1.0):
    logits = np.array(logits, dtype=float)
    scaled = logits / T
    shifted = scaled - scaled.max()  # numerical stability
    exps = np.exp(shifted)
    return exps / exps.sum()`}
