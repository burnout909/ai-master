<StageSection id="intuition" title="① Intuition">

During autoregressive decoding, the transformer generates one token at a time. Each new token must attend to every prior token, which requires storing the key (K) and value (V) projections for all past positions. This KV-cache grows with sequence length, number of layers, and — crucially — the number of attention heads. In standard Multi-Head Attention (MHA) with h heads, you store h separate (K, V) pairs per token per layer, each of dimension d_head. For a 65B-parameter model serving a 4096-token context, this can reach several gigabytes — more than the weights themselves at fp16.

Multi-Query Attention (MQA) proposed by Shazeer (2019) solves this by making all h query heads share a single (K, V) pair. The KV-cache shrinks by a factor of h, yielding dramatic speedups. The catch: training MQA from scratch sometimes degrades quality, and you cannot easily convert an existing MHA checkpoint without retraining from scratch.

**Grouped-Query Attention (GQA)** splits the difference. Instead of 1 or h KV heads, it uses g groups (1 ≤ g ≤ h), each group sharing one (K, V) pair among h/g query heads. When g = h you recover MHA; when g = 1 you recover MQA. The key insight: you can initialize a GQA model by **mean-pooling** the pretrained MHA K and V projection matrices within each group — giving a warm start that requires only ~5% of the original pretraining tokens to recover most of the original quality.

<StageSection id="math" title="② Math">

The KV-cache size for a single sequence scales as:

<Math display expr="\text{KV-cache bytes} = g \times d_\text{head} \times 2 \times L \times T \times b_e" />

where g is the number of KV groups, d_head is the head dimension, the factor 2 accounts for both K and V, L is the number of layers, T is the current sequence length, and b_e is bytes per element (2 for fp16). The reduction ratio from MHA to GQA is simply h/g: replacing h independent KV heads with g shared ones cuts the cache by that factor.

For GQA attention itself, each query head q_i in group k attends using the single shared key and value for that group:

<Math display expr="\text{Attn}(Q_i, K_k, V_k) = \text{softmax}\!\left(\frac{Q_i K_k^\top}{\sqrt{d_\text{head}}}\right) V_k, \quad k = \left\lfloor \frac{i \cdot g}{h} \right\rfloor" />

The mean-pool initialization for a group containing head indices S_k is:

<Math display expr="\hat{W}^K_k = \frac{1}{|S_k|} \sum_{j \in S_k} W^K_j, \quad \hat{W}^V_k = \frac{1}{|S_k|} \sum_{j \in S_k} W^V_j" />
