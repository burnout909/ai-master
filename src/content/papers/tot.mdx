<StageSection id="intuition" title="① Intuition">

Chain-of-Thought prompting asks a language model to reason step-by-step before answering. This works well for many tasks — but it commits the model to a single linear trajectory through the reasoning space. Once an early step goes wrong, the chain cannot recover: every subsequent step builds on that mistake, and by the end the model has wasted its entire token budget going in the wrong direction. For hard problems — the Game of 24, creative writing with structural constraints, or mini-crosswords — a wrong early branch dooms the entire attempt.

Tree of Thoughts (ToT) breaks this commitment. Instead of one chain, the model explicitly maintains a *tree* of partial reasoning states. At each step, a **thought generator** samples `k` candidate continuations (using the same LLM with an appropriate prompt). An **evaluator** — again the same LLM, but prompted differently — scores each candidate thought: "does this partial state look promising?" A **search algorithm** (BFS, DFS, or best-first) then decides which branches to expand next and which to prune. The result is deliberate, structured exploration of the reasoning space rather than a one-shot commit.

<StageSection id="math" title="② Math">

At a node `v` with partial solution `s`, the thought generator samples `k` candidate children:

<Math display expr="\{c_1, \ldots, c_k\} \sim p_{\theta}^{\text{thought}}(\cdot \mid s)" />

Each child is scored by the evaluator:

<Math display expr="V(c_i) = p_{\theta}^{\text{value}}(\cdot \mid s,\, c_i)" />

The frontier `F` is a priority queue of unexpanded nodes. At each iteration, the search selects the top-`T` nodes by score (beam search) or the single top node (best-first) and expands them. Define the beam at depth `d` as:

<Math display expr="B_d = \operatorname{top}_T\!\bigl\{\, V(v) : v \in F,\; \text{depth}(v) = d \,\bigr\}" />

Expanding `B_d` generates `k \cdot T` new candidates at depth `d+1`, each immediately scored, before the beam advances to depth `d+1`. DFS instead replaces best-first with a LIFO stack, ignoring `V` except for pruning hopeless branches.
