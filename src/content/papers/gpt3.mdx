<StageSection id="intuition" title="① Intuition">

GPT-2 showed that a large decoder-only language model could do useful things zero-shot. GPT-3 pushed that idea to an extreme: **175 billion parameters** trained on **300 billion tokens** drawn from Common Crawl, WebText2, Books, and Wikipedia. At this scale, something qualitatively new emerges — **in-context learning**.

In-context learning means the model can "learn" a new task purely from examples placed in its prompt, with **zero gradient updates**. Show it two English→French pairs and a new English phrase, and it translates. Show it three arithmetic examples and a new equation, and it solves it. The weights never change; the "learning" is entirely a forward pass conditioned on the context.

The critical lesson from GPT-3 is that this emergent ability is **scale-dependent**. Small models (1B params) show nearly no few-shot gains. Above roughly 6B, few-shot accuracy rises sharply. At 175B, GPT-3 matches or beats fine-tuned smaller models on many NLP benchmarks without a single gradient step on those tasks.

**Interact with the widget:** start with shots=0 and log₁₀(params)=8. Observe the low baseline accuracy. Now slide log₁₀(params) up past 9.5 — accuracy rises, but slowly. Then set shots to 3 or 5 and repeat: the curve lifts noticeably only at large scale. Few-shot in-context learning unlocks at roughly 6B+ parameters (log₁₀ ≈ 9.8).

<StageSection id="math" title="② Math">

GPT-3's training objective is standard autoregressive language modeling — predict each token given all preceding tokens:

<Math display expr="L = -\sum_{t=1}^{T} \log p_\theta(x_t \mid x_1, \ldots, x_{t-1})" />

At inference time, **few-shot prompting** is pure prompt-conditioning. Given k demonstrations and a query, the model scores:

<Math display expr="p_\theta(y \mid x_1, y_1, \ldots, x_k, y_k, x_{\text{query}})" />

No parameter updates. The prompt acts as a soft specification of the task distribution.

**Scaling laws** (Kaplan et al. 2020, developed alongside GPT-3) describe how the optimal loss scales with model size N when compute budget is fixed:

<Math display expr="L(N) \approx \left(\frac{N_c}{N}\right)^{\alpha}" />

where N_c is a critical parameter count and alpha is approximately 0.076 for language modeling. Doubling parameters improves loss, but with diminishing returns. This law motivated training a single 175B model rather than many smaller models.
