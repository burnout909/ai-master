<StageSection id="intuition" title="① Intuition">

Standard attention is quadratic in sequence length — both in compute and in memory. But the memory bottleneck is subtler than it first appears. A modern GPU has two kinds of storage: High Bandwidth Memory (HBM), which on an A100 holds ~40 GB but has relatively high latency; and on-chip SRAM (shared memory), which is extremely fast but totals only ~20 MB spread across all streaming multiprocessors. When naive attention computes the N×N score matrix S = QK^T, it writes all N² values to HBM so that softmax can read them back. For N=8192 and 32-bit floats, that is 256 MB written and then re-read — a round trip to slow memory, repeated every layer.

FlashAttention (Dao et al., 2022) reframes the problem as an **IO-aware** algorithm. Instead of asking "what is the minimal flop count?" it asks "how do we minimise HBM reads and writes?" The answer is tiling: divide Q, K, and V into blocks small enough to fit in SRAM simultaneously. Within each tile, compute partial scores, maintain running statistics (max `m` and normalizer `l`) to fuse the numerically-stable online softmax, and accumulate the output O — all without touching HBM. The full N×N score matrix is never materialised. The result is mathematically exact (no approximation), 2–4× faster wall-clock, and uses O(N) instead of O(N²) HBM memory.

<StageSection id="math" title="② Math">

Standard attention computes:

<Math display expr="S = \frac{QK^\top}{\sqrt{d}}, \quad P = \text{softmax}(S), \quad O = PV" />

For a row of S with values s_1, ..., s_N, the numerically stable softmax subtracts the row maximum m = max(s_i) before exponentiation: P_i = exp(s_i - m) / sum_j exp(s_j - m). Naive attention must read the entire row to find m, then read it again to compute the exponentials — two HBM passes over N² elements.

**Online softmax** fuses these two passes into one by maintaining running statistics. Given a new block of scores S_block, update the running max m and normalizer l:

<Math display expr="m_\text{new} = \max(m,\; \max_j S_{\text{block},j})" />
<Math display expr="l_\text{new} = e^{m - m_\text{new}} \, l \;\;+\;\; \sum_j e^{S_{\text{block},j} - m_\text{new}}" />
<Math display expr="O_\text{new} = \frac{l \cdot e^{m - m_\text{new}}}{l_\text{new}} \, O \;\;+\;\; \frac{1}{l_\text{new}} \sum_j e^{S_{\text{block},j} - m_\text{new}} V_{\text{block},j}" />

The factor e^(m - m_new) rescales the old statistics whenever the running max is updated by a new block. After processing all blocks, O holds the exact same result as the two-pass softmax.

  solutionCode={`import numpy as np

def online_softmax_update(m, l, O, S_block, V_block):
    """
    m:       (B_q, 1)  running row-max
    l:       (B_q, 1)  running normalizer
    O:       (B_q, d)  running output accumulator
    S_block: (B_q, B_k) raw scores for this tile
    V_block: (B_k, d)  value tile
    returns: (m_new, l_new, O_new)
    """
    m_new = np.maximum(m, S_block.max(axis=-1, keepdims=True))
    scale = np.exp(m - m_new)                         # rescale old stats
    P_block = np.exp(S_block - m_new)                 # numerically stable
    l_new = scale * l + P_block.sum(axis=-1, keepdims=True)
    O_new = (scale * l * O + P_block @ V_block) / l_new
    return m_new, l_new, O_new`}
