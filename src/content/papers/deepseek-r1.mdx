<StageSection id="intuition" title="① Intuition">

Most reasoning models are trained with instruction-style supervised fine-tuning (SFT) followed by RLHF: first show the model thousands of worked examples, then use human preference signals to refine the outputs. DeepSeek-R1-Zero does something striking: **skip SFT entirely and train with reinforcement learning from verifiable rewards**. The reward signal is grounded — a correct-answer checker for math problems, unit tests for code — not a learned preference model that can be gamed. No human labels, no curated chain-of-thought demonstrations.

What happens when you train this way? Two things emerge that weren't explicitly programmed. First, **response length roughly doubles** over the course of training: the model discovers on its own that longer reasoning chains give more opportunity to catch errors and backtrack on hard problems. Second, **"aha moments"** appear — places where the model writes something like "wait, I made an error earlier" and then corrects itself. Self-verification emerges from the reward signal alone, not from any imitation.

<StageSection id="math" title="② Math">

DeepSeek-R1 trains with **GRPO** (Group Relative Policy Optimization), a PPO variant that avoids maintaining a separate critic network. Instead, for each prompt the algorithm samples a group of G responses, scores each with the verifiable reward, and computes a group-normalized advantage. The policy-gradient update then uses this advantage with a KL penalty to keep the new policy close to a frozen reference.

The composite reward for a single response is:

<Math display expr="r = r_{\text{accuracy}} + r_{\text{format}}" />

where `r_accuracy = 1` if the final answer is verified correct (zero otherwise), and `r_format = 1` if the response obeys the required tag structure `<think>...</think><answer>...</answer>` (zero otherwise). Both components are binary and verifiable — no learned proxy.

The group-normalized advantage for response `i` in a group of G responses sampled for the same prompt is:

<Math display expr="\hat{A}_i = \frac{r_i - \mu_{\text{group}}}{\sigma_{\text{group}}}" />

where `\mu_group = (1/G) \sum_{j=1}^G r_j` and `\sigma_group = \sqrt{(1/G)\sum_{j=1}^G (r_j - \mu_group)^2}`. This replaces the critic's value estimate entirely: the baseline is the empirical mean reward within the group.

The GRPO objective clips the ratio and adds a KL penalty to the reference policy `\pi_ref`:

<Math display expr="\mathcal{L}_{\text{GRPO}} = -\mathbb{E}\!\left[\sum_{i=1}^G \min\!\left(\rho_i \hat{A}_i,\; \text{clip}(\rho_i, 1-\varepsilon, 1+\varepsilon)\hat{A}_i\right)\right] + \beta\, D_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}})" />

where `\rho_i = \pi_\theta(o_i | q) / \pi_{\text{ref}}(o_i | q)` is the probability ratio and `\beta` controls the KL penalty strength.

  solutionCode={`import numpy as np

def group_normalized_advantages(rewards: np.ndarray, eps: float = 1e-8) -> np.ndarray:
    """
    rewards: np.ndarray of shape (B, G) — B prompts, G sampled responses each
    Returns: np.ndarray of shape (B, G) — per-group normalized advantages
    """
    mean = np.mean(rewards, axis=1, keepdims=True)   # (B, 1)
    std  = np.std(rewards,  axis=1, keepdims=True)   # (B, 1)
    return (rewards - mean) / (std + eps)`}
