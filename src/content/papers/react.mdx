<StageSection id="intuition" title="① Intuition">

A pure-reasoning LLM can hallucinate facts; a pure-tool-calling agent gets lost without reasoning. What if we interleave?

**Setup:** imagine an LLM answering "What is the population of Paris?" In Chain-of-Thought (CoT) mode it reasons entirely from memory. In Act-only mode it calls tools but without a guiding plan. ReAct alternates between *thinking* and *acting*, letting each inform the other.

<PredictThenReveal client:load predictPrompt="Predict: what's the most common failure mode of CoT-only on factual questions?">
  **Confident wrong answers from stale or missing knowledge.** CoT reasons entirely from the model's parameters — if those parameters contain outdated facts or simply never saw a particular piece of data, the model will produce a fluent, confident answer that is factually wrong. There is no grounding mechanism to catch or correct the error mid-reasoning.
</PredictThenReveal>

<StageSection id="math" title="② Math">

ReAct is formalized as a policy conditioned on an interleaved context. At each step _t_, the agent samples an action from:

<Math display expr="a_t \sim \pi_\theta(a_t \mid c_t)" />

where the context grows with every step:

<Math display expr="c_t = (o_1, a_1, \ldots, a_{t-1}, o_t)" />

The action space is extended: <Math expr="a_t \in \mathcal{A} \cup \mathcal{L}" /> where **𝒜** is the set of tool calls (search, lookup, etc.) and **ℒ** is the space of natural-language thoughts. A thought action is executed purely inside the model's context — it does not interact with any external environment.

<DerivationStep client:load
  question={{
    prompt: "Which is true about a thought action a_t ∈ ℒ?",
    options: [
      { text: "It doesn't change external state — only appends to context", correct: true, explain: "A Thought is an internal reasoning step. The environment (search engine, calculator, etc.) is untouched; the context simply grows by one entry." },
      { text: "It terminates the episode and returns the answer", correct: false, explain: "A Finish action terminates the episode. A Thought is a non-terminal reasoning step." },
      { text: "It is penalized in the loss because it wastes tokens", correct: false, explain: "ReAct does not penalize Thought tokens — they are encouraged as the mechanism for grounded reasoning." },
    ],
  }}
  revealExpr="a_t \in \mathcal{L} \implies \text{env state unchanged,} \ c_{t+1} = (c_t, a_t)"
/>

**Token cost:** ReAct uses 2–5× more tokens than Act-only because each trajectory contains interleaved Thought steps that do not directly produce observations. This is a deliberate trade-off: more tokens per query buys substantially better factual accuracy and fewer dead-end action sequences.

<SrsCard id="react-thought-action" paper="react" prompt="What happens to environment state when a_t is a Thought action (a_t ∈ ℒ)?" answer="Nothing. Thoughts are internal — they only append to context c_t. Only tool-call actions (a_t ∈ 𝒜) trigger environment interactions and produce observations." />

<StageSection id="code" title="④ Code (Python)">

Implement a minimal ReAct scaffold. The loop drives a scripted LM through Thought / Action / Finish blocks, calling a canned search tool for grounding.

<HintFade client:load
  hints={[
    "Parse with a simple regex on ^(Thought|Action|Finish):",
    "The loop appends the Observation after an Action",
    "Terminate when you see Finish and return the payload",
  ]}
  solutionCode={`import re

CANNED = {
    "france capital": "Paris",
    "paris, france population": "~2,102,650 (2023)",
}
def search(q):
    for k, v in CANNED.items():
        if k in q.lower():
            return v
    return "no results"

# Scripted LM returns a pre-canned sequence based on iteration count
SCRIPT = [
    "Thought: I need the capital of France first.",
    "Action: search(\\"France capital\\")",
    "Thought: Now fetch Paris's population.",
    "Action: search(\\"Paris, France population\\")",
    "Finish: ~2,102,650 (2023)",
]

def scripted_lm(context, _state={"i": 0}):
    out = SCRIPT[_state["i"]]
    _state["i"] += 1
    return out

def parse_block(text):
    m = re.match(r"^(Thought|Action|Finish):\\s*(.*)$", text.strip(), re.DOTALL)
    if not m:
        return ("Unknown", text)
    return (m.group(1), m.group(2))

def run(query, lm, tool):
    c = [("Query", query)]
    for _ in range(10):  # max steps
        block = lm(c)
        kind, payload = parse_block(block)
        if kind == "Finish":
            return payload
        if kind == "Action":
            m = re.match(r'(\\w+)\\("?(.*?)"?\\)', payload)
            args = m.group(2) if m else payload
            obs = tool(args)
            c.append(("Action", payload))
            c.append(("Observation", obs))
        else:
            c.append((kind, payload))
    return None`}
/>
