<StageSection id="intuition" title="① Intuition">

Training a deep network is hard because the distribution of each layer's pre-activations keeps shifting as the weights below change — the paper calls this **internal covariate shift**. As distributions drift into the tails of saturating non-linearities (tanh, sigmoid), gradients collapse and learning stalls. Pre-2015, people carefully initialized weights and used tiny learning rates.

Batch Normalization attacks the shift directly: at every layer it subtracts the minibatch mean and divides by the minibatch standard deviation, re-centering activations to roughly zero-mean unit-variance. Two learnable scalars per channel, γ and β, let the network re-scale if it needs to.

  It drifts and spreads. Mean walks off (0.2 accumulates across layers); variance blows up (1.1² per layer, ×10 layers ≈ 6.7× variance). Most activations fall in the saturated regions of a downstream non-linearity, so gradients die.
</PredictThenReveal>

<StageSection id="math" title="② Math">

Batch Normalization over a minibatch _B_ of pre-activations _z_:

<Math display expr="\mu_B = \frac{1}{m}\sum_{i=1}^{m} z_i, \quad \sigma_B^2 = \frac{1}{m}\sum_{i=1}^{m} (z_i - \mu_B)^2" />

<Math display expr="\hat{z}_i = \frac{z_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad \text{BN}(z_i) = \gamma\, \hat{z}_i + \beta" />

γ, β are learned per-feature parameters. ε is a small constant (e.g. 1e-5).

<DerivationStep client:load
  question={{
    prompt: "Why add ε inside the square root?",

      { text: "To make the output have non-zero variance", correct: false, explain: "Output variance is still controlled by γ, not ε." },
      { text: "For backward-pass differentiability", correct: false, explain: "√x is differentiable away from 0; ε just shifts you away from 0." },
    ],
  }}
  revealExpr="\hat{z}_i = \frac{z_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}"
/>

<DerivationStep client:load
  question={{
    prompt: "If BN always forces mean 0, variance 1 — why do we still need learnable γ and β?",
    options: [
      { text: "To recover the identity transform if that is what the network wants", correct: true, explain: "Setting γ = √(σ²+ε), β = μ exactly undoes BN; γ, β keep representational power." },
      { text: "To stabilize the running averages at test time", correct: false, explain: "That's handled by exponential moving averages of μ, σ²." },
      { text: "To regularize the network like dropout", correct: false, explain: "BN does regularize slightly, but that's a side effect — γ, β are about representational freedom." },
    ],
  }}
  revealExpr="\text{BN}(z) = \gamma\,\hat{z} + \beta \text{ can recover any affine shape of } z"
/>

At inference we replace μ_B, σ_B² with running averages <Math expr="\bar{\mu}, \bar{\sigma}^2" /> accumulated during training.

<StageSection id="code" title="④ Code (Python)">

Implement BatchNorm's forward pass for a 2D tensor of shape (batch, features).

<HintFade client:load
  hints={[
    "Reduce mean and variance along axis 0 (the batch axis), keeping the feature axis.",
    "Use a small epsilon inside the square root for numerical stability.",
    "γ and β should broadcast correctly — shape (features,) against (batch, features).",
  ]}
  solutionCode={`import numpy as np

def batchnorm(z, gamma, beta, eps=1e-5):
    mean = z.mean(axis=0)
    var  = z.var(axis=0)
    zhat = (z - mean) / np.sqrt(var + eps)
    return gamma * zhat + beta`}
/>
