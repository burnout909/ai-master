<StageSection id="intuition" title="① Intuition">

DDPM generates data by defining a **Markov chain** through a noise schedule: the forward process adds Gaussian noise over T steps (often 1000), and the reverse process denoises step by step. This works beautifully, but it has two costs. First, sampling requires running many sequential denoising steps, which is slow. Second, the denoising path is curved — the network must predict along a complex, high-curvature trajectory from noise to data. These curves mean the model must take many small steps to stay on track.

Flow Matching (Lipman et al., 2022) takes a radically different route. Instead of a discrete Markov chain, it directly parameterizes a **time-dependent vector field** v(x, t) that continuously transports particles from a base distribution (standard Gaussian at t=0) to the data distribution (at t=1). During training, the target vector field is computed from a constructed **conditional path** — specifically, a straight line from a noise sample x_0 to a data sample x_1. Straight paths have constant velocity (x_1 - x_0), which is trivially simple to compute and minimizes kinetic energy. At inference you simply integrate the ODE from t=0 to t=1 — and because the paths are nearly straight, far fewer steps are needed.

<StageSection id="math" title="② Math">

The generative model is defined by the **probability-flow ODE**:

<Math display expr="\frac{dx}{dt} = v_\theta(x,\, t), \quad t \in [0, 1]" />

Starting from x_0 ~ N(0, I), integrating this ODE to t=1 produces a sample from the learned distribution. The network v_theta is trained to match the **marginal vector field** u_t(x), which is the average velocity at each point (x, t) under the marginal probability path p_t.

The **flow matching loss** is:

<Math display expr="\mathcal{L}_{\text{FM}} = \mathbb{E}_{t \sim U[0,1],\; x \sim p_t(x)}\!\left[\|\, v_\theta(x, t) - u_t(x)\,\|^2\right]" />

The marginal u_t is intractable (it requires integrating over all data points). The key insight of **conditional flow matching** is that training on individual conditional paths is equivalent: for a straight-line path conditioned on a source-target pair (x_0, x_1), the conditional velocity is simply:

<Math display expr="u_t(x \mid x_0, x_1) = x_1 - x_0" />

The conditional path is `x(t) = (1-t) x_0 + t x_1`. Substituting into dx/dt gives the constant vector x_1 - x_0. This makes the regression target constant across all t — an enormous simplification compared to DDPM's t-varying noise prediction.

  solutionCode={`import numpy as np

def flow_matching_loss(v_net, x0, x1, t):
    """
    v_net: callable(x, t) -> array (N, d)
    x0:   (N, d) source samples from N(0, I)
    x1:   (N, d) data samples
    t:    float in [0, 1], current time
    returns: scalar MSE loss
    """
    target = x1 - x0                    # constant conditional velocity
    x_t = (1 - t) * x0 + t * x1        # interpolated point
    pred = v_net(x_t, t)                # network prediction
    return np.mean((pred - target) ** 2)`}
