<StageSection id="intuition" title="① Intuition">

By 2023, large language models had shown remarkable capabilities in reasoning and generation — but they had a fundamental blind spot: they were frozen in time, unable to perform reliable arithmetic, and incapable of deterministic lookups. A model trained on data from 2021 has no knowledge of 2022 events. Ask it to compute 17% of 3,847 and it will hallucinate a plausible-sounding but wrong answer. Ask it for today's exchange rates and it fabricates stale numbers with confident prose.

The naive fix — fine-tune the model on tool-augmented data — requires expensive human annotation of when and how to call each tool. Toolformer sidesteps this entirely using **self-supervision**: the model itself generates candidate tool-call insertions into plain text, executes the tools to get results, and keeps only those insertions that measurably reduce the loss on the tokens that follow. The surviving insertions become training data. No human ever labels a single API call. The model teaches itself, from its own predictions about what information would be useful.

<StageSection id="math" title="② Math">

For each position `i` in a text `x` and a candidate tool call `c` with arguments, the model executes the tool to get result `r`. Define two loss values over the continuation tokens `x_{i+1:n}`:

<Math display expr="L_-(x) = \sum_{j=i+1}^{n} -\log p_\theta(x_j \mid x_{1:i})" />

This is the loss on the continuation **without** the tool call — conditioning only on the original prefix.

<Math display expr="L_+(x;\,c,r) = \sum_{j=i+1}^{n} -\log p_\theta(x_j \mid x_{1:i},\, [c \to r],\, x_{i+1:j-1})" />

This is the loss on the continuation **with** the tool call and its result inserted. The insertion `[c -> r]` is placed at position `i`, so the model conditions on both the original prefix and the tool's output.

The **filtering criterion** keeps the insertion if and only if:

<Math display expr="L_-(x)\;-\;L_+(x;\,c,r)\;\geq\;\tau" />

where `tau` is a threshold hyperparameter (set to 1.0 in the paper). Intuitively: if the tool result reduces the cross-entropy on the continuation by at least `tau` nats, the call was useful and the example is kept for fine-tuning.

  solutionCode={`import numpy as np

def toolformer_filter(l_minus: np.ndarray, l_plus: np.ndarray, tau: float) -> np.ndarray:
    """
    l_minus: 1-D array of continuation losses WITHOUT tool call, shape (n_candidates,)
    l_plus:  1-D array of continuation losses WITH tool call and result, shape (n_candidates,)
    tau:     scalar threshold; paper uses 1.0
    Returns: boolean mask of shape (n_candidates,), True where the call is useful
    """
    return (l_minus - l_plus) >= tau`}
