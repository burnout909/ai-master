<StageSection id="intuition" title="① Intuition">

DDPM (Ho et al., 2020) proved that diffusion models can generate high-quality images — but there is a catch. The forward and reverse processes both operate directly on pixel arrays. At 512x512 resolution, that means working with 512 x 512 = 262,144 individual values per image. Every U-Net forward pass during training and every denoising step at inference time must process all those values. Attention scales quadratically in the number of tokens, so moving from 256x256 to 512x512 multiplies the attention cost by 16x. Pushing to 1024x1024 is almost impractical in pixel space.

Rombach et al. (2022) asked: do we actually need to run diffusion on raw pixels? Most of the redundancy in a natural image lives at the perceptual level — the fine-grained texture details — not at the semantic level. A powerful image encoder (the encoder half of a VAE) can compress a 512x512 image into a 64x64 latent grid — 64 times fewer spatial positions — while preserving everything semantically meaningful. **Latent Diffusion Models** (LDMs) run the entire diffusion process in this compressed latent space and only decode back to pixels once, at the very end. This is the architecture behind Stable Diffusion.

<StageSection id="math" title="② Math">

LDM training happens in two phases. Phase 1 trains the VAE to learn a perceptually compressed latent representation. Phase 2 trains the diffusion model inside that latent space, with optional text conditioning via cross-attention.

**Phase 1 — VAE training:**

<Math display expr="\mathcal{L}_{\text{VAE}} = \|x - D(E(x))\|^2 + \lambda_{\text{KL}}\, \mathrm{KL}(q(z|x)\,\|\,p(z))" />

Here E is the encoder that maps image x to latent distribution q(z|x), D is the decoder that reconstructs x from a sample z, and the KL term regularises the latent space toward a standard Gaussian. A small lambda_KL (around 1e-6) provides just enough regularisation without over-compressing the latent.

**Phase 2 — LDM training:**

<Math display expr="\mathcal{L}_{\text{LDM}} = \mathbb{E}_{z_0, \varepsilon, t}\!\left[\|\varepsilon - \varepsilon_\theta(z_t, t, c)\|^2\right]" />

where z_t is the noisy latent at timestep t:

<Math display expr="z_t = \sqrt{\bar{\alpha}_t}\, z_0 + \sqrt{1 - \bar{\alpha}_t}\,\varepsilon, \quad \varepsilon \sim \mathcal{N}(0, I)" />

This is exactly the DDPM objective — except x_0 is replaced by z_0 (the VAE-encoded latent) and the U-Net now additionally receives a conditioning signal c (e.g., a text prompt encoded by CLIP). The conditioning is injected through **cross-attention layers** inside the U-Net:

<Math display expr="Q = W_Q z,\quad K = W_K c,\quad V = W_V c" />
<Math display expr="\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d}}\right) V" />

The latent feature map z attends to the text embedding c: each spatial position in z queries which text tokens are most relevant, then aggregates their value vectors. This is how "a cat sitting on a bench" steers the denoised output toward cats and benches.

  solutionCode={`import numpy as np

def cross_attention(z, c, W_Q, W_K, W_V):
    """
    z: (N, d)   latent query features
    c: (M, d_c) text key/value features
    W_Q: (d, d)   query projection
    W_K: (d_c, d) key projection
    W_V: (d_c, d_v) value projection
    returns: (N, d_v)
    """
    d = W_Q.shape[1]
    Q = z @ W_Q          # (N, d)
    K = c @ W_K          # (M, d)
    V = c @ W_V          # (M, d_v)
    scores = Q @ K.T / np.sqrt(d)   # (N, M)
    scores -= scores.max(axis=1, keepdims=True)  # numerical stability
    w = np.exp(scores)
    w /= w.sum(axis=1, keepdims=True)            # softmax
    return w @ V                                 # (N, d_v)`}
