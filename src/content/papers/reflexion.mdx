<StageSection id="intuition" title="① Intuition">

Traditional reinforcement learning improves an agent by updating its neural network weights via gradients: the policy receives a reward signal, backpropagates through millions of parameters, and nudges its behavior for the next episode. This works beautifully when you own the model. But what if your agent is a black-box LLM accessed through an API — or any frozen model where gradient-based updates are impossible or prohibitively expensive? The weights are off limits. You cannot retrain a 100B-parameter model after every maze run.

Reflexion's answer is **verbal reinforcement**: keep the weights frozen and instead let the agent update its *behavior* through language. After a failed trial, the agent — or a separate reflection module — writes a natural-language post-mortem ("the wall is at (2,2), I should navigate around it rather than through it") and stores it in an **episodic memory buffer**. The next trial begins with this memory prepended to the prompt, effectively conditioning the policy on its own accumulated hindsight. No gradients. No weight updates. The agent improves by reading its own notes.

<PredictThenReveal client:load predictPrompt="Before using the widget: imagine an agent navigating a grid with a hidden wall. On trial 1 it fails. On trial 2 it reads its own reflection ('the wall is at (2,2)'). Do you expect trial 2 to succeed? Why might it still fail, even with a perfect reflection?">
  The reflection gives the agent a crucial piece of information it lacked — the wall location — so trial 2 is more likely to avoid that specific obstacle. But verbal reinforcement is not guaranteed: the reflection must be correctly generated, the agent must attend to it and act on it, and the corrected path must not encounter *new* obstacles. Reflexion succeeds when the reflection accurately diagnoses the failure mode and the next trial's task is within the agent's in-context reasoning ability.
</PredictThenReveal>

<StageSection id="math" title="② Math">

Reflexion formalizes three interacting modules. The **Actor** is the policy `pi` — an LLM that produces actions `a_t` from the current state `s_t` and the memory buffer `M`. The **Evaluator** computes a reward `r_t` from the completed trajectory — this can be binary (success/failure) or a scalar. The **Self-Reflection** module generates a natural-language feedback string from the trajectory and reward.

At trial `t`, the loop is:

<Math display expr="\tau_t = \bigl(s_0,\,a_1,\,s_1,\,\ldots,\,a_T,\,s_T\bigr) \sim \pi(\cdot \mid s_0,\, M_{t-1})" />

The actor unrolls a trajectory `tau_t` by sampling actions from the policy conditioned on the initial state and the current memory. Then:

<Math display expr="r_t = \text{Evaluator}(\tau_t)" />

<Math display expr="\ell_t = \text{Reflector}(\tau_t,\; r_t)" />

<Math display expr="M_t = M_{t-1} \cup \{\ell_t\}" />

The memory grows by one reflection per trial. The policy at trial `t+1` conditions on `M_t`. There is no explicit gradient anywhere in this loop — the only "learning" is the expansion of `M`.

    reflect:   callable(trajectory, reward) -> str
